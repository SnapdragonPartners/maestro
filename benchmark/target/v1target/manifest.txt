# Prompt-bearing inputs of the v1 target, hashed into the MPH P dimension.
# Directory entries (trailing /) include every file beneath them.
#
# Classification rule: a file goes here when it contains text authored for
# a model — role prompts, request prompts, coder instructions, and tool
# descriptions/usage docs (tool definitions are sent to the LLM). File
# grain is deliberate: implementation edits inside these files move P
# conservatively; prompt changes can never miss it.
#
# Guarded two ways by prompthash_test.go: every entry must exist in the
# target checkout, and the independent scanner sweep (phrase + structural
# heuristics) must classify every candidate into exactly one of this file
# or allowlist.txt.
